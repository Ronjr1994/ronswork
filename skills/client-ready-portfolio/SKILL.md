# Client-Ready Portfolio Skill

## Trigger conditions

Use this skill when a user asks to create, redesign, audit, modernize, or productionize a personal portfolio, service website, freelancer site, or small-agency website.

## Required workflow

1. Define audience, pain points, promise, services, proof, and conversion goal.
2. Build the information architecture before styling.
3. Create one coherent visual system.
4. Design stable final states before adding animation.
5. Label public work, client work, internal work, and concept studies accurately.
6. Convert the main CTA into a guided brief with an email fallback.
7. Add privacy warnings and opt-in local saving.
8. Make the experience usable without external AI or backend services.
9. Add metadata, favicon, social preview, privacy page, sitemap, robots file, and validation.
10. Deliver a clean ZIP with `index.html` at the root.

## Trust rules

- Never invent metrics, testimonials, clients, or deployment status.
- Do not call a browser-based rules engine an AI chatbot.
- Use “Built with” unless a tool actually powers the live runtime.
- Require users not to submit protected health information, credentials, or confidential files.
- Keep direct-submission claims disabled until a real endpoint is configured and tested.

## Output standard

A completed project should include:

- responsive site;
- source CSS and production CSS;
- modular JavaScript;
- site configuration file;
- privacy page;
- metadata and social preview;
- reusable playbook and QA checklist;
- validation script;
- launch notes.

## Evaluation

Score the result across:

- positioning clarity;
- visual identity;
- proof quality;
- conversion path;
- trust accuracy;
- privacy;
- accessibility;
- responsiveness;
- maintainability;
- production readiness.
